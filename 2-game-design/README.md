# Phase 2 — Game Design : Les 13 PRD

Chaque aspect du jeu a son propre document de design (PRD = Product Requirement Document).

## Ordre recommandé

```
01 Core Loop ──> 02 Mécaniques ──> 03 Monde ──> 04 Progression
                                                      │
05 Économie <──────────────────────────────────────────┘
     │
     └──> 06 UI/HUD ──> 07 Social ──> 08 Items ──> 09 PNJ/Ennemis
                                                         │
10 Audio/Visuel <────────────────────────────────────────┘
     │
     └──> 11 Onboarding ──> 12 Monétisation ──> 13 Événements
```

**Pourquoi cet ordre ?**
- Le Core Loop définit tout le reste
- Les Mécaniques détaillent le Core Loop
- Le Monde, la Progression et l'Économie forment le "squelette" du jeu
- UI, Social, Items, PNJ habillent ce squelette
- Audio/Visuel donne l'atmosphère
- Onboarding, Monétisation et Événements sont les derniers car ils dépendent de tout le reste
